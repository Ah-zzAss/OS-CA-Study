# 다양한 보조기억 장치

보조기억장치에는 대표적으로 하드 디스크와 플래시 메모리가 존재한다.

### 하드 디스크

하드 디스크는 자기적인 방식으로 데이터를 저장한다.

- 플래터 : 실질적으로 데이터가 저장되는 동그란 원판
- 스핀들 : 플래터를 회전시키는 구성요소

![image.png](attachment:a92ace4d-6fe0-4a66-b656-14bfe13ede76:image.png)

- 헤드 : 플래터를 대상으로 데이터를 읽고 쓰는 바늘
- 디스크 암 : 헤드를 이동시키는 장치

![image.png](attachment:c9655649-7d32-45b8-85e0-0e8f882bf5df:image.png)

플래터는 트랙과 섹터 단위로 데이터를 저장한다.

![image.png](attachment:4db5966c-87bb-4808-970f-c270b4c760e8:image.png)

여러 겹의 플래터 상에서 같은 트랙이 위치한 곳을 모아 실린더라고 부른다.

연속된 정보는 보통 한 실린더에 기록된다. 

![image.png](attachment:331b3555-9141-44ab-a33a-5873ce701c5a:image.png)

1. 탐색 시간 (seek time)
    
    접근 하려는 데이터가 저장된 트랙까지 헤드를 이동시키는 시간

3. 전송 시간 (transfer time)

### 플래시 메모리

전기적으로 데이터를 읽고 쓸 수 있는 반도체 기반의 저장 장치

보조기억장치 뿐만 아니라 다양한 곳에서 널리 사용하는 저장 장치

데이터를 저장하는 가장 작은 단위를 셀(cell)을 사용한다. 

하나의 셀에 몇 비트를 저장 할 수 있느냐에 따라 플래시 메모리 종류가 나뉜다.

1. SLC(single level cell), 1비트
    1. 한 셀로 2개의 정보를 표현
    2. MLC, TLC에 비해 속도와 수명이 빠르고 길지만 용량 대비 가격이 높다.
2. MLC(multiple level cell), 2비트
    1. 한 셀로 4개의 정보를 표현
    2. SLC보다 속도와 수명은 떨어지지만 용량 대비 가격이 저렴하다.
3. TLC(triple level cell), 3비트
    1. 한 셀로 8개의 정보를 표현
    2. SLC, MLC보다 속도와 수명은 떨어지지만 용량 대비 가격이 저렴하다. ‘

셀들이 모여서 페이지, 페이지가 모여 블록, 블록이 모여 플레인, 플레인이 모여 다이를 이룬다.

![image.png](attachment:4420e7af-0fbc-4ba4-9322-07bd99c4e45e:image.png)

플래시 메모리에서 읽고 쓰는 것은 페이지 단위로 이루어지고, 삭제는 블록 단위로 이루어진다.

읽기/쓰기 단위와 삭제 단위가 다르다는 것이 가장 큰 특징 중 하나이다.

이때 페이지는 세 개의 상태를 가질 수 있다.

1. Free
    
    어떠한 데이터도 저장하고 있지 않아 새로운 데이터를 저장할 수 있는 상태
    
2. Valid
    
    이미 유효한 데이터를 저장하고 있는 상태
    
3. Invalid
    
    쓰레기값이라 부르는 유효하지 않은 데이터를 저장하고 있는 상태
    

하드 디스크와 달리 덮어쓰기가 불가능하여 Valid 상태인 페이지에는 새 데이터를 저장할 수 없다.

따라서 한 블록에 4개의 페이지로 이루어져 있다고 가정할 때

A,B,C가 저장되어 있고 A를 A’로 수정하려면 기존 A를 Invalid 상태로 변환하고 새로운 A’가 저장된다.

A같이 쓰레기 값을 정리하기 위해 가비지 컬렉션 기능을 사용한다.

유효한 페이지만 새로운 블록에 복사한 뒤 그 전 블록을 삭제한다.(블록단위 삭제가 가능하므로)

# RAID의 정의와 종류

### RAID의 정의

여러 개의 물리적 보조기억장치를 하나의 논리적 보조기억장치처럼 사용하는 기술

### RAID의 종류

RAID 구성 방법을 RAID 레벨이라고 표현

대표적으로 RAID 0, RAID 1, RAID 2, RAID 3 등등이 있다.

그 중 대표적인 0, 1, 4, 5, 6만 알아보겠다.

1. RAID 0
    
    여러 개의 보조기억장치에 데이터를 단순히 나누어 저장하는 방식
    
    데이터를 하드 디스크 개수만큼 나누어서 각 하드 디스크에 번갈아 가며 데이터를 저장
    
    ![image.png](attachment:0412ff88-571e-488e-b3fb-5fd29b5cbb08:image.png)
    
    줄무늬처럼 분산되어 저장된 데이터를 스트라입(stripe)이라 하고, 분산하여 저장하는 것을 스트라이핑(striping)이라고 한다.
    
    분산되어 저장되었으므로 동시에 읽고 쓸 수 있기 떄문에 빠르지만 하나의 하드 디스크에 문제가 생긴다면 다른 모든 정보에 문제가 생긴다.
    
    ![image.png](attachment:58c24242-50af-4c30-b904-10e7544f34a1:image.png)
    
2. RAID 1
    
    복사본을 만드는 방식. 미러링(mirroring)이라고도 부른다.
    
    RAID 0처럼 데이터 스트라이핑으로 저장하지만 복사본을 만든다.
    
    RAID 0에 비해서 절반의 용량 정보를 저장 할 수 있다.
    
    문제가 발생해도 잃어버린 정보를 금방 찾을 수 있다.
    
    ![image.png](attachment:f730b1a2-b999-4c20-8b9d-a36971096e05:image.png)
    
3. RAID 4
    
    RAID 1처럼 완전한 복사본을 만드는 대신 오류를 검출하고 복구하기 위한 정보인 패리티 비트(parity bit)를 두는 방식
    
    패리티 비트를 이용해 오류를 검출하고 복구할 수 있다.
    
    원래 패리티 비트는 오류 검출만 가능하지만 RAID에서는 패리티 값으로 오류 수정도 가능하다.
    
    RAID 1보다 적은 하드 디스크로도 데이터를 안전하게 보관할 수 있다.
    
    ![image.png](attachment:a432f29e-d0ba-48df-9990-c4168e80dfc4:image.png)
    
4. RAID 5
    
    RAID 4에서는 새로운 데이터가 저장될 때 마다 패리티를 저장하는 디스크에도 데이터를 쓰므로 병목 현상이 발생한다.
    
    RAID 5에서는 패리티 정보를 분산하여 저장하는 방식으로 병목 현상을 해소한다.
    
    ![image.png](attachment:bf3c66a3-13e6-459d-8c08-c5efcb917761:image.png)
    
5. RAID 6
    
    기본적으로는 RAID 5와 같으나 두 개의 패리티를 둔다.
    
    오류를 검출하고 복구할 수 있는 수단이 두 개이므로 더욱 안전하지만 쓰기 속도는 RAID 5보다 느리다. 
    
    데이터 저장 속도를 조금 희생하고 안전하게 보관하기 위한 방식
