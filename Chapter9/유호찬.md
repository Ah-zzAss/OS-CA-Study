# 운영체제를 알아야 하는 이유

### 운영체제란

모든 프로그램은 하드웨어를 필요로 한다.

프로그램 실행에 필요한 요소들을 가리켜서 **자원**(CPU, 메모리, 보조기억장치 등등)이라고 한다. 

프로그램에 필요한 자원을 할당하고 프로그램 실행을 돕는 프로그램이 **운영체제**

운영체제 또한 프로그램이기에 메모리 내에 커널 영역이라는 공간에 적재된다.

그 이에 사용자가 이용하는 응용 프로그램은 사용자 영역에 적재된다. 

![image.png](attachment:2b9410d9-d6c9-4648-9333-3b4323ecde7d:image.png)

운영체제는 프로그램을 적정한 메모리 주소에 적재하고, CPU 자원을 할당하고, 입출력장치 자원을 할당해준다.

운영체제 내부에서도 자원별로 기능이 나눠져 있다. 

ex) CPU 관리, 프로세스 관리, 파일 시스템 관리 등등

### 운영체제를 알아야 하는 이유

운영체제가 없다면 하드웨어 조작하는 코드조차 개발자가 모두 작성해야한다.

운영체제를 알면 문제 해결의 실마리를 찾을 수 있다.

문제가 생길 시 운영체제가 오류 메시지를 띄워준다.

# 운영체제의 큰 그림

### 운영체제의 심장, 커널

커널 : 프로그램이 올바르고 안전하게 실행되게 하는 기능

사용자 인터페이스 : 윈도우의 바탕화면 같이 사용자가 컴퓨터와 상호작용할 수 있는 통로(CUI, CLI 등) 

### 이중 모드와 시스템 호출

이중 모드

CPU가 명령어를 실행하는 모드를 구분

1. 사용자 모드 : 운영체제 서비스를 제공 받을 수 없는 실행 모드, 자원 접근 X
2. 커널 모드 : 운영체제 서비스를 제공 받을 수 있는 실행 모드, 자원 접근 O

시스템 호출

운영체제에게 요청을 보내 운영체제 서비스를 받기를 위함.

소프트웨어 인터럽트 : 인터럽트 처리 순서와 유사. 

1. 시스템 호출
2. 커널 모드로 변환
3. 운영체제 코드 실행
4. 시스템 호출 복귀

### 운영체제의 핵심 서비스

프로세스 관리

실행 중인 프로그램을 프로세스라 한다. 

CPU는 한 번에 하나의 프로세스만 실행할 수 있기에 여러 프로세스는 계속 번갈아가면서 실행

자원 접근 및 할당

모든 프로세스는 자원을 필요로 한다.

CPU, 메모리, 입출력장치 등의 자원을 프로세스에 할당해줌

파일 시스템 관리

파일을 열고, 생성하고, 삭제하는 작업
