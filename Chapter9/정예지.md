# Chapter9: 운영체제 시작하기

## 1. 운영체제

- **실행할 프로그램에 자원을 할당하고, 프로그램이 올바르게 실행되도록 돕는 프로그램**
- 메모리 내에 **커널 영역**에 따로 적재되어 실행되는 **프로그램**
- 사용자 인터페이스(종류: GUI, CLI): 운영체제가 제공하는 서비스 중 커널에 포함되지 않은 서비스

(사진)

> - **시스템 자원, 자원**: 프로그램 실행에 필요한 요소 (CPU, 메모리, 보조기억장치, ... )
> - **사용자 영역**: 메모리 내 사용자가 이용하는 응용프로그램이 적재되는 영역


## 2. 이중 모드

CPU가 명령어를 실행하는 모드를 **사용자 모드, 커널 모드로 구분**하는 방식

- **시스템 콜 (시스템 호출)** 를 이용하여 커널 모드로 전환한다.
> - **시스템 콜**: 사용자 모드로 실행되는 프로그램이 **운영체제 서비스를 제공 받기 위해 커널 모드로 전환**하기 위한 **요청(소프트웨어 인터럽트)**

### (1) 사용자모드
운영체제 서비스를 제공받을 수 **없는**, 즉 커널 영역의 코드를 실행할 수 없는 모드

### (2) 커널 모드
**운영체제 서비스를 제공받을 수 있는** 실행 모드

- 운영체제는 커널 모드로 실행한다.
- CPU가 ***커널 모드로 명령어를 실행하면*** 자원에 접근하는 명령어 외에도 모든 명령어 실행 가능

## 3. 핵심 서비스

운영체제는 다음 3가지 핵심 서비스를 제공한다.

1. 프로세스 관리
2. 자원 접근 및 할당
3. 파일 시스템 관리

### (1) 프로세스 관리
운영체제는 다양한 상태의 프로세스를 관리한다.

- **배경**: 하나의 CPU는 한 번에 하나의 프로세스만 실행 가능하기에 CPU는 프로세스들을 조금씩 번갈아가며 실행한다.
- **고려 사항**:
    - **프로세스 동기화**: 여러 프로세스가 동시에 실행되는 환경
    - **교착 상태**

### (2) 자원 접근 및 할당
운영체제는 CPU, 메모리, 보조기억장치와 입출력장치에 접근하고 조작함으로서 프로세스에 필요한 자원을 할당한다.

#### 운영체제가 접근하는 자원들
- **CPU**: CPU 스케줄링을 통해 공정하게 CPU를 할당
- **메모리**: 메모리 할당, 메모리가 부족할 경우 대처
- **입출력장치**: 커널 영역의 인터럽트 서비스 루틴을 통해 입출력 작업 실행
