# 명령어

## 고급 언어와 저급 언어

**고급 언어**: 프로그래밍 언어와 같이 컴퓨터가 이해하는 언어가 아닌 사람이 이해하고 작성하기 쉽게 만들어진 언어이다.

**저급 언어** : 컴퓨터가 직접 이해하고 실행할 수 있는 언어로 명령어로 이루어져 있다. 기계어와 어셈블리어 두 가지 종류가 있다.

(사진)

- 기계어: 0과 1의 명령어 비트로 이루어진 언어이다. 기계어는 오로지 컴퓨터만을 위해 이루어진 언어이기 때문에 사람이 읽으면 의미를 이해하기 어렵다.
- 어셈블리어: 0과 1로 표현한 명령어르 이루어져 있다.

## 컴파일 언어와 인터프리터 언어

### 컴파일 언어

컴파일 방식으로 작동하는 프로그래밍 언어이다.

컴파일러에 의해 소스코드 전체가 저급 언어로 변환되어 실행되는 고급 언어이다.

컴파일이 성공적으로 수행되면 개발자가 작성한 소스 코드는 컴퓨터가 이해할 수 있는 저급 코드로 변환되며 이를 `목적 코드`라고 한다.

(사진)

### 인터프리터 언어

인터프리트 방식으로 작동하는 프로그래밍 언어이다.

소스 코드를 한 줄씩 차례로 실행한다. 소스 코드를 한 줄씩 실행하기 때문에 소스 코드 전체를 저급 언어로 변환하는 시간을 기다릴 필요가 없다.

소스 코드 내에 오류가 하나라도 있으면 컴파일이 불가능했던 컴파일 언어와는 달리 소스 코드에 문법 오류가 있더라도 문법 오류 전 줄까지 수행된다.

일반적으로 인터프리터 언어는 컴파일 언어보다 느린데, 인터프리터 언어는 소스 코드 마지막에 이를 때까지 한 줄씩 저급 언어로 해석하며 실행해야 하기 때문이다.

## 연산 코드와 오퍼랜드

명령어는 연산 코드와 오퍼랜드로 구성되어 있다.

**연산 코드**: 명령어가 수행할 연산을 말하며, 연산자라고도 부른다.

**오퍼랜드**: 연산에 사용할 데이터가 저장된 위치를 말하며, 피연산자라고도 부른다.

(사진)

연산 코드가 담기는 영역을 연산 코드 필드, 오퍼랜드가 담기는 영역을 오퍼랜드 필드라고 한다.

### 오퍼랜드

연산에 사용할 데이터 또는 연산에 사용할 데이터가 저장된 위치를 의미한다. 따라서, 오퍼랜드 필드에는 숫자와 문자 등을 나타내는 데이터 또는 메모리나 레지스터 주소가 올 수 있다.

주로 연산에 사용할 데이터가 저장된 위치가 담기기 때문에 오퍼랜드 필드를 주소 필드라고 부르기도 한다.

(사진)

오퍼랜드가 하나도 없는 명령어를 `0-주소 명령어`, 하나인 명령어를 `1-주소 명령어`, 두 개인 명령어를 `2-주소 명령어`, 세 개인 명령어를 `3-주소 명령어`라고 부른다.

### 연산 코드

명령어가 수행할 연산을 의미한다.

종류가 매우 많지만 기본적인 연산 코드 유형은 크게 네 가지로 나눌 수 있다.

- 데이터 전송
    - MOVE, STORE, LOAD, PUSH, POP 등
- 산술/논리 연산
    - ADD, SUBTRACT, MULTIPLY, DIVIDE, AND, OR, NOT, COMPARE 등
- 제어 흐름 변경
    - JUMP, CONDITIONAL JUMP, HALT, CALL, RETURN 등
- 입출력 제어
    - READ, WRITE, START IO, TEST IO 등

## 주소 지정 방식

오퍼랜드 필드에 메모리 주소가 담긴다면 표현할 수 있는 데이터의 크기가 하나의 메모리 주소에 저장할 수 있는 공간만큼 커지기 때문에 메모리나 레지스터의 주소를 담는다.

연산 코드에 사용할 데이터가 저장된 위치, 즉 연산의 대상이 되는 데이터가 저장된 위치를 `유효 주소`라고 한다.

오퍼랜드 필드에 데이터가 저장된 위치를 명시할 때 연산에 사용할 데이터 위치를 찾는 방법을 `주소 지정 방식`이라고 한다.

### 즉시 주소 지정 방식

(사진)

연산에 사용할 데이터를 오퍼랜드 필드에 직접 명시하는 방식이다. 

표현할 수 있는 데이터의 크기가 작아지는 단점이 존재하지만 매우 빠르다는 장점이 존재한다.

### 직접 주소 지정 방식

(사진)

오퍼랜드 필드에 유효 주소를 직접적으로 명시하는 방식이다.

즉시 주소 지정 방식보다 오퍼랜드 필드에서 표현할 수 있는 데이터의 크기가 커졌으나 표현할 수 있는 오퍼랜드 필드의 길이가 연산 코드의 길이만큼 짧아져 표현할 수 있는 유효 주소에 제한이 있다.

### 간접 주소 지정 방식

(사진)

유효 주소의 주소를 오퍼랜드 필드에 명시한다.

직접 주소 지정 방식보다 표현할 수 있는 유효 주소의 범위는 넓어졌으나 두 번의 메모리 접근이 필요해 속도가 느리다.

### 레지스터 주소 지정 방식

(사진)

직접 주소 지정 방식과 비슷하게 연산에 사용할 데이터를 저장한 레지스터를 오퍼랜드 필드에 직접 명시하는 방법이다.

CPU 내부에 있는 레지스터에 접근하는 방식이 메모리에 접근하는 것보다 빠르기 때문에 직접 주소 방식보다 빠르게 데이터에 접근이 가능하다.

다만, 표현할 수 있는 레지스터 크기에 제한이 생길 수 있다.

### 레지스터 간접 주소 지정 방식

(사진)

연산에 사용할 데이터를 메모리에 저장하고, 그 주소를 저장한 레지스터를 오퍼랜드 필드에 명시하는 방법이다.

유효 주소를 찾는 과정이 간접 주소 지정 방식과 비슷하나 메모리에 접근하는 횟수가 한 번으로 줄어든다.
