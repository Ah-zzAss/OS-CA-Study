### 컴파일 언어와 인터프리터 언어

1. 컴파일 언어
    1. 고급 언어(소스 코드)를 컴파일러가 컴파일 과정을 통해 목적 코드(저급 언어)로 변환
    2. 소스 코드 전체를 훑어보고 저급 언어로 컴파일
    3. 중간에 오류를 하나라도 발견하면 컴파일 실패
2. 인터프리터 언어
    1. 인터프리터에 의해 고급 언어(소스 코드)가 한 줄씩 실행
    2. 소스 코드 N번째 줄에 오류가 있더라고 N-1번째 줄까지는 올바르게 수행

### 실행 파일

1. 컴파일 과정을 거쳐서 만들어진 목적 코드에서 링킹이라는 작업을 거쳐 실행 파일이 만들어짐.
2. 링킹 : 목적 코드에 없는 외부 기능들을 연결 짓는 작업

1. 연산 코드 
    1. 명령어가 수행할 연산

    3. 크게 데이터 네 가지 유형으로 나뉨.

        4. 입출력 제어 : READ, WRITE 등
2. 오퍼랜드
    1. 연산에 사용할 데이터가 저앙된 위치
    2. 피연산자, 주소필드라고 부름
    3. 오펀랜드의 갯수에 따라서 0-주소명령어, 1-주소명령어, 2-주소명령어, 3-주소명령어라고 부름

### 주소 지정 방식

연산 코드의 길이와 오퍼랜드의 갯수에 따라서 하나의 오퍼랜드 필드로 표현 할 수 있는 정보의 가짓수가 줄어듦.

유효 주소를 찾는 주소 지정 방식을 통해 표현할 수 있는 데이터의 크기를 늘릴 수 있다.  

1. 즉시 주소 지정 방식
    1. 오퍼랜드 필드에 사용할 데이터를 직접 명시하는 방식 
    2. pros - 데이터를 찾는 과정이 없으므로 빠름
    3. cons - 표현하는 데이터의 크기가 작아짐
2. 직접 주소 지정 방식
    1. 유효 주소를 직접적으로 명시하는 방식
    2. pros - 메모리의 크기만큼 표현할 수 있는 데이터의 크기가 커짐
    3. cons - 여전히 오퍼랜드의 비트 수 만큼 유효 주소 제한이 있음
3. 간접 주소 지정 방식
    1. 유효 주소의 주소를 오퍼랜드 필드에 명시
    2. pros - 유효 주소의 범위가 더 늘어남
    3. cons - 두 번의 메모리 접근이 필요하므로 느림
4. 레지스터 주소 지정 방식
    1. 연산에 사용할 데이터를 저장한 레지스터를 오퍼랜드 필드에 직접 명시
    2. pros - CPU 내부에 있는 레지스터에 접근하므로 빠름
    3. cons - 직접 주소 지정 방식과 동일하게 표현할 수 있는 레지스터 크기에 제한이 생김
5. 레지스터 간접 주소 지정 방식
    1. 연산에 사용할 데이터는 메모리에 저장하고 해당 주소를 저장한 레지스터를 오퍼랜드 필드에 명시
    2. pros - 간접 주소 지정 방식과 비슷하지만 메모리에 1번 접근하여 빠름
