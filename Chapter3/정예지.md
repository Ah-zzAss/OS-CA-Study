# Chapter3: 소스 코드와 명령어

우리가 작성한 프로그래밍 언어는 컴퓨터가 이해하고 실행할 수 있는 0,1의 언어로 변환되어야 한다.
즉 고급 언어가 저급 언어, 명령어로 변환되어야 소스 코드가 실행될 수 있다.
- 고급언어: 프로그래밍 언어
- 저급 언어: 컴퓨터가 이해하고 실행할 수 있는 언어

## 1. 저급언어

### 1. 기계어
0과 1로 이루어진 명령어 모음

### 2. 어셈블리어
기계어를 읽기 편한 형태로 번역한 언어
컴퓨터가 어떤 절차로 실행되는지 추적, 관찰이 가능하다.

<center>

| 기계어 | 어셈블리어 |
| :--- | :--- |
| 0101 0101 | push rbp |
| 0101 1101 | pop rbp |
</center>

## 2. 고급언어를 저급언어로 변환하는 방식

### 1. 컴파일
컴파일러를 통해 소스코드 전체를 저급언어로 변환하는 과정

- 소스코드 내에서 오류가 하나라도 발생하면 실패
- 컴파일 결과로 저급언어로 변환된 '목적 코드'를 얻게됨

> **목적 파일**  
> 목적 코드로 이루어진 파일
> 실행 파일이 되기 위해서는 기능과 기능을 호출한 부분을 연결짓는 '링킹'이라는 작업이 필요하다.

### 2. 인터프리트

인터프리터로 소스 코드를 한 줄씩 실행하는 과정

- N번째 줄에 문법 오류가 존재해도 그 전 줄까지는 올바르게 수행됨
- 인터프리터 언어는 한줄씩 저급 언어로 해석하며 실행해야하므로 컴파일 언어보다 느리다.

<center>

| 변환 방식 | 해당 방식으로 작동하는 프로그래밍 언어 | 대표 언어 |
| :--- | :--- | :--- |
| 컴파일 | 컴파일 언어 | C |
| 인터프리트 | 인터프리터 언어 | python |

</center>

## 3. 명령어의 구조

<center>

|  |  |
| :--- | :--- |
| operation code (연산자) | operand (피연산자)|


</center>

저급언어인 기계어, 어셈플리어 또한, 명령어이기 때문에 명령어의 구조를 가집니다.

### 1. Operand (피연산자)

연산에 사용할 데이터 or 연산에 사용할 데이터가 저장된 위치를 의미합니다.  
명령어 안에 하나도 없을 수도, 여러개 존재할 수도 있습니다.

- 사용할 데이터를 명시한다.
- (대부분) 데이터가 저장된 주소 위치를 명시한다.

> 주소를 담는 경우가 많기 때문에 Operand 필드를 주소 필드라고 부르기도 합니다.

### 2. operation code (연산자)

명령어가 수행할 연산을 의미합니다.

**[기본적인 연산 코드 유형]**
- 데이터 전송
- 산술/논리 연산
- 제어 흐름 변경
- 입출력 제어

## Operand 에서 주소 지정 방식

연산에 사용할 데이터를 찾는 방법을 말한다.  
현대 CPU 가 사용하는 다양한 주소 지정 방식은 다음과 같다.

### 1. 즉시 주소 지정 방식

- 연산에 사용할 데이터를 operand 필드에 직접 명시
- 데이터를 찾을 필요가 없기 때문에 가장 빠르다

### 2. 직접 주소 지정 방식

- 연산에 사용할 데이터의 유효한 주소를 operand 필드에 명시
- 유효 주소를 표현할 수 있는 범위가 연산 코드의 비트 수를 제외한 크기로 제한됨

### 3. 간접 주소 지정 방식

- 연산에 사용할 데이터의 유효한 주소의 주소를 operand 필드에 명시
- 두 번의 메모리 접근이 필요해서 앞서 설명한 방식보다 일반적으로 느리다

### 4. 레지스터 주소 지정 방식

- 연산에 사용할 데이터를 저장한 레지스터를 operand 필드에 직접 명시
- CPU 내부 레지스터에 접근하여 외부 메모리 접근 보다 빠르다
- 표현할 수 있는 레지스터 크기에 제한이 있을 수 있다.

### 5. 레지스터 간접 주소 지정 방식

- 연산에 사용할 데이터를 메모리에 저장하고, 데이터를 저장한 메모리의 주소를 레지스터에 저장한다.  
    그 주소를 저장한 레지스터를 operand 필드에 명시한다.
- 메모리에 접근하는 횟수는 1번으로, [3. 간접 주소 지정 방식]보다 빠르다
