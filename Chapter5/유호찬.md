# 빠른 CPU를 위한 설계 기법

### 클럭

CPU의 속도 단위

클럭 속도가 높은 CPU는 일반적으로 성능이 높다.

### 코어와 멀티코어

코어는 명령어를 실행하는 부품

멀티 코어는 코어를 여러 개 퐘하는 CPU를 말한다.

### 스레드와 멀티스레드

스레드에는 하드웨어적 스레드와 소프트웨어적 스레드가 있다.

1. 하드웨어적 스레드 : 하나의 코어가 동시에 처리하는 명령어 단위.
2. 소프트웨어적 스레드 : 하나의 프로그램에서 독립적으로 실행되는 단위

하나의 명령어를 실행하기 위해 꼭 필요한 레지스터인 레지스터 세트가 1코어에 여러 개 있다면 1코어에서 여러 개의 명령어가 동시에 처리 가능해진다.

멀티스레드 프로세서는 하나의 코어로 여러 개의 명령어를 동시에 실행할 수 있는 CPU를 말한다.

프로그램 입장에서는 하드웨어 스레드가 4개면 CPU가 네 개 있는 것처럼 보인다.

그래서 하드웨어 스레드를 논리 프로세서라고 부르기도 한다.

### 명령어 파이프라이닝

명령어가 처리 단계에서 같은 단계가 겹치지 않도록 동시에 실행하는 방법

<img width="842" height="547" alt="image" src="https://github.com/user-attachments/assets/7a6781ef-890e-4944-b570-d412fa71bdcd" />

파이프라인 위험 - pipeline hazard

1. 데이터 위험 - data hazard
    
    데이터 의존성으로 인해 발생하는 문제.
    이전 명령어가 끝나야만 다음 명령어를 수행할 수 있는 경우 동시에 실행이 불가
    
2. 제어 위험 - control hazard
    
    프로그램 카운터의 갑작스러운 변화(분기)로 인하여 미리 처리하던 명령어가 쓸모없어지는 경우
    이를 위해 분기 예측(branch prediction)이 사용된다.
    
3. 구조적 위험 - structural hazard
    
    명령어를 동시에 실행하는 과정에서 같은 CPU 부품을 사용하려고 할 때 발생

### 슈퍼스칼라

CPU 내부에 여러 개의 명령어 파이프라인을 포함한 구조

여러 개의 파이프라인을 사용하면 pipeline hazard를 피하기 더욱 까다롭기 때문에 파이프라인 개수에 비례하여 빨라지지는 않는다.

# CISC와 RISC

### 명령어 집합

CPU가 이해할 수 있는 명령어들의 모음을 명령어 집합 구조(ISA : Instruction Set Architecture)라고 한다.

CPU마다 ISA가 다를 수 있다. ISA가 같은 CPU끼리는 서로의 명령어를 이해할 수 있지만 다르면 이해하지 못한다. 즉 ISA는 일종의 CPU의 언어이다.

### CISC - Complex Instruction Set Computer

복잡한 명령어 집합을 활용하는 컴퓨터라는 의미

1. 복잡하고 다양한 명령어
2. 가변 길이의 명령어 사용
3. 다양한 주소 지정 방식
4. 상대적으로 적은 명령어로도 프로그램 실행 가능
5. 파이프라이닝 하기 어려

### RISC -Reduced Instruction Set Computer

적은 명령어 집합을 활용하는 컴퓨터라는 의미

1. 단순하고 적은 명령어
2. 고정 길이의 명령어 사용
3. 적은 주소 지정 방식
4. 프로그램을 이루는 명령어의 수가 많음
5. 파이프라이닝 하기 쉬움
