# Chapter5: CPU 성능 향상 기법

## 1. CPU 성능을 위한 구성

CPU의 성능을 높이기 위해선 3가지 방법이 있다.
1. 클럭 속도 높이기
2. CPU 코어 높이기
3. 스레드 수 늘리기

### 1. 클럭

CPU의 속도 단위

- **클럭 속도**는 **1초에 클럭이 몇 번** 반복되는지, 헤르츠 단위로 측정한다.
- 일정한 속도를 가지지 않는다.
- 고성능을 요구하는 경우에는 순간적으로 속도를 끌어올리는 **오버클럭킹**을 한다.

### 2. 코어

명령어를 실행하는 부품

- CPU를 여러 개 포함하고 있는 CPU는 **멀티코어 CPU** 혹은 **멀티코어 프로세서**라고 부른다.
- 반드시 코어 수에 비례하여 CPU의 연산 속도가 증가하지 않는다.

### 3. 스레드

실행 흐름의 단위

#### 하드웨어적 스레드

CPU에서 사용되는 스레드
하나의 코어가 동시에 처리하는 명령어 단위

- 하나의 코어로 여러 명령어를 동시에 처리하는 CPU를 **멀티스레드 프로세서**, **멀티스레드 CPU**라고 한다.
  (참고) 하이퍼스레딩은 인텔의 멀티스레드 기술을 부르는 용어이다.
- 하드웨어 스레드를 논리 프로세서라고 부르기도 한다.
  (프로그램 입장에서 2코어 4스레드의 경우 4개의 명령어를 동시에 처리 가능하므로, 논리프로세서가 4개이다.)

#### 소프트웨어적 스레드

프로그램에서 사용되는 스레드
하나의 프로그램에서 독립적으로 실행되는 단위

- 여러 기능을 작동 시키는 코드도 각각의 스레드로 만들면 동시에 실행할 수 있다.  
    (1코어 1스레드 CPU가 여러 스레드로 만들어진 프로그램을 실행할 수 있다.)

## 2. 명령어 병렬 처리 기법

CPU가 한시도 쉬지 않고 작동되어야 높은 성능을 보여줄 수 있다.

### 1. 명령어 파이프라인

명령어들을 명령어 파이프라인에 넣고 동시에 처리하는 기법

- 명령어는 `인출 -> 해석-> 실행-> 결과 저장` 처리 단계를 거친다.
- 명령어는 같은 단계가 겹치지 않는다면 동시에 실행 가능하다.

- **슈퍼스칼라**: CPU 내부에 여러 개의 명령어 파이프라인을 포함한 구조
- **슈퍼스칼라 프로세서, 슈퍼 스칼라 CPU**: 슈퍼스칼라 구조로 명령어 처리가 가능한 CPU (ex, 멀티 스레드 프로세서)
  - 매 클럭 주기마다 한 번에 여러 개의 명령어 인출, 실행 가능
  - 이론적으로는 파이프라인 개수에 비례하여 속도가 빨라지지만, 파이프라인 위험 같은 문제를 생각하면 꼭 그렇지도 않다.

### 2. 파이프라인 위험

**파이프라이닝 중 성능 향상에 실패하는 경우**를 말한다.

파이프라인 위험에는 3가지가 있다.
1. 데이터 위험
2. 제어 위험
3. 구조적 위험

#### 데이터 위험

데이터 의존적인 두 명령어를 무작정 동시에 실행하려고 할 때 파이프라인이 제대로 작동하지 않는 것
- 원인: 명령어간 데이터 의존성

#### 제어 위험

기본적으로 프로그램 카운터는 현재 실행 중인 명령어의 다음 주소로 갱신된다.
프로그램 실행 흐름이 바뀌어 명령어가 실행되며 프로그램 카운터 값에 갑작스러운 변화가 생길 때 기존에 해당 값을 가지고와서 파이프라인에서 처리중이었던 명령어는 무쓸모가 되는 것

- 원인: 분기 등으로 인한 프로그램 카운터의 갑작스러운 변화
- 분기 예축: 프로그램이 어디로 분기할지 미리 예측한 후 그 주소를 인출하는 기술

#### 구조적 위험 (=자원 위험)

서로 다른 명령어가 동시에 같은 CPU 부품 (ALU, 레지스터)를 사용하려고 할 때 발생하는 것

### 3. 비순차적 명령어 처리 (Out-of-order excution, OoOE)

명령어의 합법적인 새치기, 명령어들을 순차적으로 실행하지 않는 기법

### 3. CISC와 RISC
