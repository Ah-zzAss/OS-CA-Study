# Chapter6: 메모리와 캐시 메모리

## 1. RAM의 특징과 종류

### 특징

- 실행할 프로그램의 명령어, 데이터가 저장된다.
- 전원을 끄면 모두 날아가는 휘발성 저장 장치이다.
- 보조 기억 장치(비휘발성)에는 보관할 데이터, RAM(휘발성)에는 실행할 대상을 저장
- 용량이 크다면 실행속도가 어느정도 증가하지만 무조견 비례하지는 않음

    - 프로그램 하나만 저장할 수 있을 정도로 작다면 다른 프로그램을 실행하고 싶을 때마다 보조기억장치에서 가지고 와야함
    - 프로그램 여러개를 저장할 수 있다면 미리 가져와서 저장할 수 있고, 동시에 실행할 수 있음

### 종류

#### (1) DRAM

- Dynamic RAM, 저장된 데이터가 동적으로 사라짐
- 데이터의 소멸을 막기위해 주기적으로 재활성화 필요
- 우리가 주로 사용 -> 소비 전력이 비교적 낮고, 저렴, 집적도가 높아 대용량으로 설계 가능하기때문

#### (2) SRAM

- Static RAM, 저장된 데이터가 변하지 않음
- DRAM 보다 일반적으로 속도가 빠름
- 주로 캐시 메모리로 사용
- 소비전력이 크고, 비쌈, 집적도가 낮아 빽빽하게 설계하기 어려움

#### (3) SDRAM

- 클럭 신호와 동기화된 DRAM
- 클럭 타이밍에 맞춰 CPU와 정보를 주고 받을 수 있음
- SDR(Single Data Rate) RAM 이라 부르기도

#### (4) DDR SDRAM

- 최근 가장 흔히 사용되는 RAM
- 대역폭 (데이터를 주고 받는 길의 너비) 을 넓혀 속도를 빠르게 만든 SDRAM
- SDRAM은 한 클럭에 데이터 1개를 주고 받지만 DDR SDRAM는 데이터 2개를 주고 받음
- DDR2 SDRAM(4배), DDR3 SDRAM(8배), DDR4 SDRAM(16배)

## 2. 메모리의 주소 공간

### 주소 종류
- 물리 주소: 정보가 실제로 저장된 하드웨어 상의 주소
- 논리 주소:
    - CPU와 실행 중인 프로그램이 사용하는 주소
    - 실행 중인 프로그램에게 각각 부여된 주소로, 0번지 부터 시작된다.
    - 프로그램 시작점으로부터 떨어진 거리

### 메모리관리 장치(MMU)

- **위치**: CPU와 주소 버스 사이
- **역할**: 논리 주소를 물리주소로 변환
- **동작**: 
    - CPU가 발생시킨 논리 주소에 베이스 레지스터 값을 더하여 논리주소를 물리주소로 변환
    - CPU가 한계 레지스터보다 높은 논리 주소에 접근하려고 하면 인터럽트(트랩)을 발생 시킴

> **MMU를 구성하는 핵심 레지스터**
> - 베이스 레지스터: 프로그램의 첫 물리 주소를 저장
> - 한계 레지스터: 논리 주소의 최대 크기를 저장

## 3. 캐시 메모리

`CPU가 연산하는 속도 > CPU 가 메모리에 접근하는 시간` => 이 간극을 메우기 위한 것이 캐시 메모리

### (1) 저장 장치 계층 구조

CPU와 얼마나 가까운가를 기준으로 계층적으로 나눈 구조

### (2) 캐시 메모리

레지스터보다 용량이 크고 메모리보다 빠른 **SRAM 기반의 저장장치**

- **위치**: CPU와 메모리 사이
    - 코어와 가장 가까운 순으로 L1(코어 내부), L2(코어 내부), L3(코어 외부)캐시
- **역할**: CPU의 연산 속도와 메모리 접근 속도의 차이를 줄이기
- **동작**: CPU 내에 메모리가 필요할때 L1 -> L2 -> L3 캐시 순으로 확인

> **멀티 코어 프로세서**: L1-L2 캐시는 코어마다 고유한 캐시 메모리로 할당, L3은 여러 코어가 공유하는 형태로 사용

### (3) 참조 지역성 원리

캐시 메모리는 CPU가 자주 사용할법한 대상을 예측하여 저장, 저장하는 원리가 바로 참조 지역성 원리

1. CPU는 최근에 접근했던 메모리 공간에 다시 접근하려는 경향이 있다 (시간 지역성)
2. 최근에 접근했던 메모리 공간 근처를 접근하려는 경향이 있다 (공간 지역성)

> **캐시 적중률  = 캐시 히트 횟수 / (캐시 히트 횟수 + 캐시 미스 횟수)**
> - 캐시 히트 : 캐시 메모리 내 데이터가 CPU에서 활용될 경우
> - 캐시 미스: 캐시 메모리 내에 저장했지만, 예측이 틀려 직접 메모리에서 데이터를 가져와야하는 경우
