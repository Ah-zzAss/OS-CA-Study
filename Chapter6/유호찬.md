# RAM의 특징과 종류

### RAM의 특징

1. 전원을 끄면 저장된 명령어와 데이터가 모두 날아감 - 휘발성 저장 장치
2. 전원을 꺼도 내용이 저장되는 보조기억장치(SSD, USB 등)에 저장하고 실행할 때 RAM으로 복사한 뒤 실행

### RAM의 용량과 성능

RAM의 용량이 커질수록 실행 시간이 짧아짐

RAM의 용량이 충분히 크다면 보조기억장치에서 많은 데이터를 미리 저장할 수 있음

### RAM의 종류

1. DRAM 
    1. 저장된 데이터가 동적으로 변하는 RAM
    2. 흔히 사용하는 RAM
2. SRAM 
    1. 저장된 데이터가 변하지 않는 RAM
    2. DRAM보다 속도도 더 빠름
    3. 소비전력이 크고 가격이 비싼 단점이 있음
3. SDRAM
    1. 클럭 신호에 맞춰 동작하며 CPU와 정보를 주고받을 수 있는 DRAM
4. DDR SDRAM
    1. 대역폭을 넓혀 속도를 빠르게 만든 SDRAM
    2. 대역폭 - 데이터를 주고받는 길의 너비
    3. 최근 가장 흔히 사용되는 RAM
    4. DDR2, DDR3, DDR4 등 더 넓은 SDRAM이 존재함

# 메모리의 주소 공간

### 물리주소와 논리주소

1. 물리 주소
    1. 메모리 하드웨어가 사용하는 주소
2. 논리 주소
    1. CPU와 실행 중인 프로그램이 사용하는 주소

<img width="1575" height="671" alt="스크린샷 2026-04-05 222251" src="https://github.com/user-attachments/assets/6c81ace9-1d6e-410c-a8b5-ff8918dbbc01" />

1. 메모리 관리 장치(MMU)
    1. 논리 주소와 물리 주소 간 변환을 수행
    2. 베이스 레지스터 값에 CPU가 발생시킨 논리 주소를 더하여 물리 주소로 변환

### 메모리 보호 기법

해당 프로그램의 논리 주소 영역을 벗어나는 명령어가 실행되면 안되므로 한계 레지스터가 존재함

- 한계 레지스터
    - 논리 주소의 최대 크기를 저장
    - 프로그램의 물리 조소 범위는 베이스 레지스터 ~ (베이스 레지스터 + 한계 레지스터)
    - 한계 레지스터보다 높은 논리 주소에 접근하면 인터럽트 발생

# 캐시 메모리

### 저장 장치 계층 구조

1. CPU와 가까운 저장 장치는 빠르다.
2. 속도가 빠른 저장 장치는 용량이 작고 비싸다.

<img width="840" height="399" alt="스크린샷 2026-04-05 223412" src="https://github.com/user-attachments/assets/af4da1b0-ddb5-43c7-bcb6-d63ebabb703a" />

### 캐시 메모리

1. CPU 레지스터와 메모리(RAM)사이에 위치한 SRAM 기반의 저장 장치
2. 레지스터와 메모리 사이의 속도 차이를 줄이기 위함
3. 캐리 메모리들은 L1, L2, L3 캐시 메모리로 이루어져 있음
4. L1이 가장 비싸고 빠르며 용량이 작다.

### 참조 지역성 원리

1. 캐시 적중률(cache hit ratio) 
    1. 캐시 메모리는 CPU가 사용할 법한 대상을 예측하여 저장한다. 
    2. 실제로 맞는 경우를 캐시 히트(cache hit)
    3. 틀려서 메모리에서 직접 가져와야 하는 경우를 캐시 미스(cache miss)
    4. 캐시 적중률(cache hit ratio) = 캐시 히트 횟수 / (캐시 히트 횟수 + 캐시 미스 횟수)
2. 참조 지역성의 원리
    1. 시간 지역성 : CPU는 최근에 접근했던 메모리 공간에 다시 접근하려는 경향
    2. 공간 지역성 : CPU는 접근한 메모리 공간 근처를 접근하려는 경향
    3. 두 가지 참조 지역성의 원리를 이용하여 CPU가 사용할 법한 데이터를 예측한다.
