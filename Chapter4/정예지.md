# Chapter4: CPU의 작동 원리

## 1. ALU

CPU의 주요 구성요소 중 하나인 ALU는 **연산**을 담당한다.

- 레지스터에서 피연산자를 읽어오고,  
제어장치에서 제어 신호를 받아와 연산 코드를 수행한다.

- 연산 결과 값을 레지스터에 저장,  
플래그를 플래그 레지스터에 저장한다.

> **플래그** : 연산에 대한 추가적인 상태 정보
> 
> - 이 플래그는 **플래그 레지스터에** 저장된다.
> - **대표적인 플래그:** 부호 플래그, 계산 결과가 0인지 여부를 나타내는 제로 플래그, 올림수/빌림수 발생 여부를 나타내는 캐리 플래그, 오버플로우 플래그, 인터럽트 가능 여부를 나타내는 인터럽트 플래그, 슈바이저 플래그

## 2. 제어장치

제어장치는 **제어 신호를 내보내고 명령어를 해석**하는 부품이다.

1. 제어장치는 **클럭 신호**를 받아들인다.  
    ***클럭**: 컴퓨터의 모든 부품들이 기준 삼아 움직일 수 있도록 하는 시간 단위

2. 제어장치는 수행할 **명령어, 플래그**를 받아들인다.
3. 제어 장치는 **제어 신호**를 받아들이거나 **CPU 내부/외부로 제어신호를** 보낸다.
4. 제어 장치의 동작 과정
    > 1. 명령어 레지스터로부터 해석할 명령어를 받아들임
    > 2. 명령어를 해석한 뒤
    > 3. 플래그 레지스터로부터 플래그 값을 받아들임
    > 4. 플래그 값을 참고하여 제어신호를 보낸다.

## 3. 레지스터

대표적인 레지스터 8가지를 알아봅시다.

<img width="775" height="432" alt="image" src="https://github.com/user-attachments/assets/39b6366f-262d-4bdc-b9df-ddfd4319bd81" />

1. **프로그램 카운터 (명령어 포인터)**
    - 메모리에서 가져올 명령어의 주소

2. **명령어 레지스터:**
    - 메모리에서 읽어들인 명령어를 저장하는 레지스터
    - 제어 장치는 명령어 레지스터 속 명령어를 받아들이고, 이를 해석한 뒤 제어 신호를 내보냄

3. **메모리 주소 레지스터:**
    - 메모리 주소를 저장하는 레지스터
    - CPU가 읽으려하는 주소값을 주소 버스로 보낼 때 여길 거쳐간다

4. **메모리 버퍼 레지스터:**
    - 메모리와 주고 받을 값을 저장하는 레지스터
    - 메모리에 쓸 값 or 메모리에서 전달 받은 값이 여길 거쳐간다.

5. **범용 레지스터**
    - 데이터, 주소 모두 저장할 수 있는 레지스터

6. **플래그 레지스터**
    - ALU 연산 결과에 따른 플래그를

7. **스택 포인터**
    - 스택에 마지막으로 저장된 값의 위차를 저장하는 레지스터

8. **베이스 레지스터**
    - 기준 주소 값을 저장하는 레지스터

## 4. 명령어

### (1) 명령어 사이클

하나의 명령어가 처리되는 주기

<img width="775" height="434" alt="image" src="https://github.com/user-attachments/assets/f7e275ee-01a1-4ed7-b273-2bb6ea9293c5" />

1. 인출 사이클: 메모리의 명령어를 CPU로 가져옴
2. 실행 사이클: CPU로 가져온 명령어를 실행

- 실행 사이클로 가기 전 명령어를 실행하기 위해 메모리 접근이 한 번 더 이뤄지는 경우 '간접 사이클'을 거친다.

### (2) 인터럽트

CPU의 정상적인 작업을 방해하는 신호를 의미

#### 동기 인터럽트 (예외)

CPU에 의해 발생하는 인터럽트
(ex, 프로그래밍 상의 오류)

#### 비동기 인터럽트 (하드웨어 인터럽트)

입출력 장치에 의해 발생하는 인터럽트

**[처리 순서]**
1. 입출력 장치가 CPU에 인터럽트 요청 신호를 보낸다
2. 실행 사이클이 끝나고 명령어 인출 전 항상 인터럽트 여부 체크
3. CPU는 인터럽트 요청 확인,
    인터럽트 플래그를 통해 현재 인터럽트를 받아들일 수 있는지 체크
4. CPU는 인터럽트 벡터를 참조하여 인터럽트 서비스 루틴을 실행
5. 인터럽트 서비스 루틴 실행 후 4에서 백업한 작업을 복구, 실행 재개

> - **인터럽트 요청 신호**: 인터럽트 전 CPU에게 보내는 신호
> - **인터럽트 플래그**: 하드웨어 인터럽트 수용 여부를 결정
    (우선순위가 높은 정전, 하드웨어 고장으로 인한 인터럽트는 수용 여부 결정 불가, 무조건 수용)
> - **인터럽트 서비스 루틴 (인터럽트 핸들러)**: 메모리에 존재하는 인터럽트를 처리하기 위한 프로그램
> - **인터럽트 벡터**: 인터럽스 서비스 루틴의 시작 주소를 저장

- 인터럽트 발생 시 프로그램 카운터의 값이 '인터럽트 벡터'로 수정됨, 스택 영역에 백업 메모리에 원래 실행하려던 주소 값 데이터가 존재
