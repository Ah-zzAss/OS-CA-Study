### ALU

CPU 내부에서 계산을 담당

<img width="729" height="388" alt="스크린샷 2026-03-27 170519" src="https://github.com/user-attachments/assets/0e67ab6b-5674-48a1-ae6b-fd1093434566" />

받아들이는 정보
  1. 레지스터 → 피연산자
  2. 제어장치 → 제어신호

내보내는 정보
  1. 결과값 : 메모리가 아닌 레지스터에 저장(숫자, 문자, 메모리 주소 등등)
  2. 플래그 : 연산 결과에 대한 추가적인 상태 정보를 플래그 레지스터에 저장(부호, 제로, 캐리, 오버플로우, 인터럽트, 슈퍼바이저 등)

### 제어장치

명령어를 읽어 들이고, 해석하고, 제어 신호를 내보내는 부품

<img width="859" height="528" alt="스크린샷 2026-03-27 170603" src="https://github.com/user-attachments/assets/f1b83398-6a7e-4247-b02d-3dece9353ff3" />

받아들이는 정보
  1. 클럭 신호 : 클럭은 컴퓨터의 모든 부품을 움직일 수 있게 하는 시간 단위
  2. 해석해야 할 명령어 : 명령어 레지스터로부터 명령어를 받아들임
  3. 플래그 값 : 플래그 레지스터 속 플래그 값을 받아들임
  4. 제어 신호 : 시스템 버스의 제어 버스로부터 전달된 제어 신호를 받아들임
    
내보내는 정보
  1. CPU 외부에 전달하는 제어 신호 : 제어 버스로 제어 신호를 내보냄
      1. 메모리
      2. 입출력장치 : 보조기억장치 포함
  2. CPU 내부에 전달하는 제어 신호
      1. ALU
      2. 레지스터

# 레지스터

CPU안에 다양한 레지스터가 있고 각기 다른 역할을 가지고 있다.

<img width="717" height="404" alt="스크린샷 2026-03-27 170649" src="https://github.com/user-attachments/assets/2def86b2-5960-4b22-b7f0-169442469113" />

1. 프로그램 카운터 : 메모리에서 가져올 명령어의 주소를 저장
2. 명령어 레지스터 : 메모리에서 읽어 들인 명령어를 저장, 제어 장치가 명령어 레지스터에서 명령어를 바다들임
3. 메모리 주소 레지스터 : 메모리의 주소를 저장
4. 메모리 버퍼 레지스터 : 메모리와 주고받을 값을 저장
5. 범용 레지스터 : 말 그대로 범용으로 사용하는 레지스터
6. 플래그 레지스터 : 플래그를 저장
7. 스택 포인터 : 스택의 꼭대기(마지막으로 저장한 값)를 가리키는 레지스터

### 특정 레지스터를 이용한 주소 지정 방식

### 스택 주소 지정 방식 (스택 포인터)

스택 영역 : 메모리에서 스택처럼 사용할 영역

스택 영역과 스택 포인터를 이용하여 주소를 지정하는 방식

가장 최근에 저장한 값이 가장 위에 존재하고 해당 메모리 위치를 스택 포인터가 가리킨다.

### 변위 주소 지정 방식(프로그램 카운터, 베이스 레지스터)

1. 상대 주소 지정 방식 : 오퍼랜드와 프로그램 카운터의 값을 더하여 유효 주소를 얻는 방식
2. 베이스 레지스터 주소 지정 방식 : 오퍼랜드와 베이스 레지스터의 값을 더하여 유효 주소를 얻는 방식

# 명령어 사이클과 인터럽트

### 명령어 사이클

CPU가 프로그램을 실행할 때 프로그램 속 명령어들은 일정한 주기를 반복되며 실행하는데 이 주기를 명령어 사이클이라고 한다.

기본적으로 인출 사이클고 실행 사이클이 존재하고 
메모리 접근이 더 필요한 경우 간접 사이클과
인터럽트가 발생한 경우 인터럽트 사이클까지 존재한다.

### 인터럽트

동기 인터럽트 : CPU에 의해 발생하는 인터럽트
  1. 예외라고 부른다.

비동기 인터럽트 : 입출력 장치에 의해 발생하는 인터럽트. 
  1. 하드웨어 인터럽트라고 부른다. 알림과 같은 인터럽트이다. 비동기 인터럽트를 받기 전까지 CPU는 다른 작업을 처리할 수 있다.
  2. 하드웨어 인터럽트 처리 순서
      1. 입출력장치가 CPU에 인터럽트 요청 신호를 보낸다.
      2. CPU가 실행 사이클이 끝나고 명령어를 인출하기 전 인터럽트 여부를 확인
      3. CPU가 인터럽트 플래그를 통해 현재 인터럽트를 받아들일 수 있는지 체크
      4. 가능하다면 현재까지의 작업을 백업(스택 영역에)
      5. 인터럽트 벡터를 참조하여 인터럽트 서비스 루틴을 실행
      6. 인터럽트 서비스 루틴이 끝나면 백업해둔 작업을 복구하여 다시 실행
  3. 용어
      1. 인터럽트 요청 신호 : 인터럽트하기 전 CPU에게 보내는 신호
      2. 인터럽트 플래그 : CPU가 인터럽트 수용 가능한지 여부 확인
      3. 인터럽트 서비스 루틴 : 인터럽트를 처리하기 위한 프로그램
      4. 인터럽트 벡터 : 수많은 인터럽트 서비스 루틴 중 식별하기 위한 정보
