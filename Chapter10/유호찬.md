# 프로세스 개요

프로그램은 실행 전, 보조기억장치에서는 데이터 덩어리 이지만 메모리에 적재되고 실행하는 순간 그 프로그램은 프로세스가 된다. 

프로세스는 사용자가 볼 수 있는 공간에서 실행되는 포그라운드 프로세스와 보이지 않는 공간에서 실행되는 백그라운드 프로세스가 있다.

백그라운드 프로세스는 유닉스에서는 데몬(daemon), 윈도우에서는 서비스(service)라고 한다.

### 프로세스 제어 블록

프로세스는 실행을 위해 CPU를 필요로 한다.

각 프로세스는 정해진 시간만큼 CPU를 이용하고 시간이 끝나는 타이머 인터럽트가 발생하면 다음 차례를 기다린다. 

이러한 실행 수서를 관리하기 위해 프로세스 제어 플록(PCB)를 이용한다.

PCB : 프로세스와 관련돈 정보를 저장하는 자료 구조

PCB는 프로세스가 실행될 때 커널 영역에 만들어지고 실행이 끝나면 폐기된다.

PCB의 정보

1. 프로세스 ID(PID) : 프로세스를 식별하기 위한 고유 번호
2. 레지스터 값 : 실행하며 사용했던 레지스터 값들을 저장
3. 프로세스 상태 : 현재 프로세스의 상태
4. CPU 스케줄링 정보 : 프로세스가 언제 어떤 순서로 CPU를 할당 받을지
5. 메모리 관리 정보 : 프로세스가 메모리에 어느 주소에 저장되어 있는지에 대한 정보
6. 사용한 파일과 입출력장치 목록 : 프로세스 실행 과정 중 사용한 입출력장치, 파일 정보 

### 문맥 교환

문맥 : 하나의 프로세스 수행을 재개하기 위해 기억해야 할 정보, PCB에 저장되어 있는 정보 

기존 프로세스의 문맥을 PCB에 백업하고 새로운 프로세스를 실행하기 위해 문맥을 PCB로 부터 복구하여 새로운 프로세스를 실행하는 것을 문맥 교환이라고 한다.

### 프로세스의 메모리 영역

하나의 프로세스에서 PCB는 커널 영역에 생성된다.

사용자 영역에는 크게 코드 영역, 데이터 영역, 힙 영역, 스택 영역으로 나뉘어 저장된다.

코드와 데이터 영역은 크기가 고정되어 있는 정적 할당 영역

힙과 스택 영역 크기가 변하는 동적 할당 영역

1. 코드 영역
    
    텍스트 영역이라고도 부른다.
    
    말 그래도 실행할 수 있는 코드, 즉 기계어로 이루어진 명령어가 저장된다.
    
    데이터가 아닌 CPU가 실행할 명령어가 담겨 있기 때문에 쓰기가 금지되어 있다. 읽기 전용
    
2. 데이터 영역 
    
    실행되는 동안 유지할 데이터가 저장되는 공간. ex) 전역 변수
    

1. 힙 영역
    
    프로그램을 만드는 사용자가 직접 할당할 수 있는 저장 공간.
    
    할당 했다면 언젠가는 해당 공간을 반환해야함.
    
    반환하지 않는다면 메모리 누수가 발생
    
2. 스택 영역
    
    데이터가 일시적으로 저장하는 공간. ex) 지역 변수
    
    PUSH와 POP
    
    일반적으로 힙 영역은 낮은 주소부터, 스택은 높은 주소부터 할당된다.
    

# 프로세스 상태와 계층 구조

### 프로세스 상태

1. 생성 상태
    
    프로세스를 생성 중인 상태
    
    이제 막 메모리에 적재되어 PCB 할당받은 상태
    
2. 준비 상태
    
    CPU를 할당 받아 실행할 수 있지만 차례를 기다리고 있는 상태
    
3. 실행 상태
    
    CPU를 할당받아 실행 중인 상태
    
4. 대기 상태
    
    실행 도중 입출력장치를 사용하는 경우가 있다. 입출력 장치는 CPU에 비해 느리므로 입출력장치가 작업을 끝날 떄 까지( 인터럽트를 받을 때 까지) 기다림
    
5. 종료 상태
    
    프로세스가 종료된 상태 
    
    운영체제는 종료된 프로세스의 PCB와 메모리를 정리한다. 
    

### 프로세스 계층 구조

프로세스는 실행 도중 시스템 호출로 인해 다른 프로세스를 생성할 수 있다.

부모 프로세스 : 새 프로세스를 생성한 프로세스

자식 프로세스 : 부모 프로세스에 의해 생성된 프로세스

다른 프로세스이기에 다른 PID를 가지지만, 일부 운영체제에서는 부모의 주소인 PPID도 저장된다. 

### 프로세스 생성 기법

복제와 옷 갈아입기 

fork() = 복제, exec() = 옷 갈아입기

fork를 통해서 자신의 복사본을 만들고 exec를 통해 복사본의 메모리 공간을 새로운 프로그램으로 덮어쓰는 시스템 호출이다.

# 스레드

실행의 단위, 더 자세히 말하자면 프로세스를 구성하는 실행의 흐름 단위

하나의 프로세스는 여러 개의 스레드를 가질 수 있고 여러 부분을 동시에 실행할 수 있다.

### 프로세스와 스레드

단일 스레드 프로세스 : 하나의 프로세스는 한 번에 하나의 일만을 처리

멀티 스레드 프로세스 : 하나의 프로세스가 여러 일을 동시에 처리

스레드는 각자의 스레드ID, 프로그램 카운터 값을 비롯한 레지스터 값, 스택으로 구성된다.

실행에 필요한 최소한의 정보만을 유지하고 프로세스 자원을 공유하며 스레드가 실행된다.

### 멀티프로세스와 멀티스레드

멀티프로세스 : 여러 프로세스를 동시에 실행하는 것

멀티스레드 : 여러스레드로 프로세스를 동시에 실행하는 것 

단일 스레드 프로세스를 여러개 vs 하나의 프로세스를 여러 스레드

프로세스는 기본적으로 자원을 공유하지 않지만 스레드끼리는 같은 프로세스 내의 자원을 공유

프로세스를 fork하면 모든 자원이 복제되어 메모리에 적재되어 낭비된다.

스레드는 각기 다른 스레드 ID, 프로그램 카운터 값을 포함한 레지스터 값, 스택을 가질뿐 프로세스의 자원을 공유한다.

메모리를 더 효율적으로 사용한다.
